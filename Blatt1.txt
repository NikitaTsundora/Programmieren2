Gradle
Legen Sie in der Konsole ein neues Gradle-Projekt für eine Java-Applikation an (ohne IDE!). Das Build-Script soll in Groovy erzeugt und als Test-API soll JUnit5 oder JUnit6 verwendet werden.
    • gradle init 
Wie finden Sie auf der Konsole heraus, welche Tasks es gibt? Erklären Sie das Projektlayout, d.h. wo kommen beispielsweise die Java-Dateien hin?
    • Mit Hilfe von gradle tasks kann man sich alle möglichen tasks ausgeben lassen.
    • src/main/java -> Code 
    • src/test/java -> Tests 
Erklären Sie, in welche Abschnitte das generierte Buildskript unterteilt ist und welche Aufgaben diese Abschnitte jeweils erfüllen. Gehen Sie dabei im Detail auf das Plugin application und die dort bereitgestellten Tasks und deren Abhängigkeiten untereinander ein.
    • plugins -> Fügt zusätzliche Funktionen in Gradle hinzu
    • repositories -> Definiert, woher externe Bibliotheken geladen werden 
    • dependencies -> Verwaltung aller Abhängigkeiten (Libraries) 
    • application -> Definiert die Startklasse (main-Methode) und wird benötigt, um die App zu starten
    • test -> Aktiviert JUnit 5 (Jupiter Plattform) 
    • Plugin application: 
            ▪ run -> Startet die Applikation
            ▪ startScripts -> Erstellt betriebssystemspezifische Startskripte (Windows und Linux), um die Anwendung als JVM-Programm auszuführen
            ▪ InstallDist -> Installiert die Anwendung in ein Zielverzeichnis (lokal ausführbare Version mit allen nötigen Dateien)
            ▪ distZip -> Erstellt ein vollständiges ZIP-Archiv der Anwendung (JAR-Datei, benötigte Bibliotheken, Startskripte)
            ▪ distTar -> Erstellt ein TAR-Archiv (ähnlich wie ZIP) 
    • https://docs.gradle.org/current/userguide/application_plugin.html

Das Spotless-Plugin wird in Gradle eingebunden und konfiguriert, um Java-Code automatisch zu formatieren. Mit spotlessCheck kann geprüft werden, ob der Code korrekt formatiert ist, und mit spotlessApply wird der Code automatisch in ein einheitliches Format gebracht. Eine Demo zeigt den Unterschied zwischen unformatiertem und automatisch formatiertem Java-Code.
